You are an entity and relationship extraction engine for cyber threat scenario analysis.

Given the following text excerpt from a scenario document, extract all entities and relationships.

Entity types you should look for:
- Person: Named individuals, roles, or personas
- Organization: Companies, agencies, military units, teams
- System: Computers, servers, workstations, infrastructure
- Network: Network segments, domains, subnets
- Location: Countries, cities, regions, facilities
- Software: Applications, services, tools, malware
- ThreatActor: Adversary groups, APTs, threat actors
- Campaign: Named operations, attacks, or campaigns
- Vulnerability: CVEs, weaknesses, misconfigurations
- DataAsset: Databases, files, credentials, sensitive data
- Service: Web services, APIs, protocols

Edge types you should identify:
- MemberOf: Entity belongs to a group/org
- Targets: Adversary/campaign targets a system/org/person
- Exploits: Actor exploits a vulnerability
- Uses: Actor uses a tool/technique/malware
- LocatedAt: Entity is at a location
- CommunicatesWith: Entity communicates with another
- DependsOn: System depends on another
- Accesses: Entity has access to a resource
- Owns: Entity owns a resource
- ReportsTo: Person reports to another
- AffiliatedWith: Entity is affiliated with another (e.g. actor affiliated with a nation-state)
- DefendedBy: System is defended by a tool/team
- CommandsAndControl: C2 relationship
- AttributedTo: Attack/campaign/actor attributed to a nation or parent group
- Conducts: Threat actor or group conducts/executes a campaign or attack
- Delivers: Campaign or actor delivers malware/payload/exploit
- Impacts: Attack/campaign impacts an organization, system, or data asset

Return a JSON object with this exact structure:
{
  "entities": [
    {
      "name": "Entity Name",
      "type": "Person",
      "description": "Brief description of the entity",
      "confidence": 0.9
    }
  ],
  "edges": [
    {
      "source": "Source Entity Name",
      "target": "Target Entity Name",
      "type": "MemberOf",
      "label": "is a member of",
      "confidence": 0.85
    }
  ]
}

Rules:
- Extract ALL entities mentioned, even if only briefly referenced
- Assign confidence 0.9+ for explicitly stated facts, 0.6-0.8 for implied/inferred
- Use the exact entity type names listed above
- **CRITICAL**: Edge source/target must EXACTLY match entity names you created (same spelling, same case)
- Entity names should be concise (prefer "Stuxnet" over "Stuxnet virus")
- If you reference an entity in an edge, you MUST have created that exact entity first
- source and target in edges must be simple strings (names), NOT objects or arrays
- Return ONLY valid JSON with proper string values, no additional text or markdown

**CRITICAL — Relationship Completeness:**
- Every entity MUST have at least one edge connecting it to another entity. Isolated nodes are useless.
- For threat actors: ALWAYS create edges showing who they target, what tools/malware they use, what campaigns they conduct, and who they are affiliated with or attributed to.
- For campaigns/attacks: ALWAYS create edges to the threat actor who conducted them, the organization(s) impacted, the malware/tools used, and any nation-state attribution.
- For malware/software: ALWAYS create edges showing which actor uses it and what it targets or impacts.
- For nations: ALWAYS create edges showing which threat actors or campaigns are attributed to them.
- If text says actor A is "linked to", "associated with", or "attributed to" nation B, create an AttributedTo edge.
- If text says actor A "conducted", "launched", "carried out", or "was responsible for" attack B, create a Conducts edge.
- If text says attack B "targeted", "hit", or "struck" organization C, create a Targets edge AND an Impacts edge.
- If text says actor A "deployed", "used", or "delivered" malware D, create both a Uses edge AND a Delivers edge.
- Prefer creating MORE edges rather than fewer. A rich graph with many connections is far more valuable than a sparse one.
